## GitLab CI/CD2

Основной файл пайплайна GitLab находится в корне репозитория: `.gitlab-ci.yml`.  
GitLab **требует**, чтобы этот файл лежал на верхнем уровне, поэтому он не перенесён в `ci/`.

В этой папке планируется хранить:

- вспомогательные скрипты для пайплайна:
  - подготовка `kubeconfig` через SSH;
  - проверка/создание SSH‑туннеля;
  - создание ConfigMap/Secret из `.prod.env` на сервере;
  - применение Kubernetes‑манифестов с retry;
- документацию по CI/CD процессу.

Пример будущей структуры:

```text
ci/
  gitlab/
    README.md
    deploy.sh              # основной сценарий деплоя (вызывается из .gitlab-ci.yml)
    helpers.sh             # общие функции: check_tunnel, check_api, apply_with_retry
```

Сейчас логика деплоя реализована внутри `.gitlab-ci.yml`.  
Постепенно она будет выноситься в `deploy.sh`/`helpers.sh`, чтобы упростить и сократить сам `.gitlab-ci.yml`.
